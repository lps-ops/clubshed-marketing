# ClubShed Marketing Site

Static marketing site for ClubShed, the equipment tracker for grassroots football clubs. Built with Astro for maximum SEO performance.

## What this is

- The public-facing site (homepage, pricing, blog, comparison pages)
- Lives at `clubshed.pro`
- The actual app lives at `app.clubshed.pro` (Lovable, separate project)

## Setup

```bash
npm install
npm run dev
```

Site runs at `http://localhost:4321`.

## Build and deploy

```bash
npm run build
```

Builds to `./dist`. Deploy this folder to Cloudflare Pages (recommended) or Netlify.

### Cloudflare Pages

1. Push this repo to GitHub
2. Cloudflare Pages → Create project → Connect to GitHub
3. Build command: `npm run build`
4. Build output directory: `dist`
5. Add custom domain: `clubshed.pro`
6. In DNS, point `app.clubshed.pro` to your Lovable project

## Architecture

```
clubshed.pro          → this site (Astro, SEO-optimized)
app.clubshed.pro      → Lovable app (the actual product)
```

The marketing site links to `app.clubshed.pro/signup` for the CTA. No deep integration needed.

## Structure

- `src/pages/index.astro` — homepage
- `src/pages/pricing.astro` — pricing page
- `src/pages/blog/` — blog posts (one per file)
- `src/layouts/BaseLayout.astro` — header, footer, SEO meta, schema
- `src/styles/global.css` — design system
- `public/` — favicons, logo SVGs, OG image, robots.txt

## SEO checklist (already done)

- Server-rendered HTML, no client-side rendering
- Per-page title and meta description
- Canonical URLs
- Open Graph and Twitter card tags (with custom OG image at /og-default.png)
- JSON-LD schema (SoftwareApplication)
- Sitemap auto-generated by `@astrojs/sitemap`
- robots.txt pointing to the sitemap
- Mobile-responsive
- No JS-blocking content

## Adding a blog post

Create a new `.astro` file in `src/pages/blog/`. Use the existing cornerstone post as a template. Then add an entry to the `posts` array in `src/pages/blog/index.astro`.
